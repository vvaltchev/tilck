# Coverage viewer — feature spec (parity target: LCOV genhtml report)

This document catalogs **100% of the functionality** of the LCOV
`genhtml` HTML report (LCOV 1.15, as produced by
`generate_coverage_report --gen` into `build/coverage_html/`). It is the
parity checklist for the `coverage-viewer` TUI: every visible element,
color, number, sort order and navigation path below must have a TUI
equivalent. Derived from the raw genhtml output, not screenshots, so the
element/color/threshold facts are exact.

## 0. Data model

genhtml renders a single LCOV tracefile (`coverage.info`). Relevant
records (per source file):

```
SF:<absolute source path>
FN:<line>,<mangled function name>      # function start line
FNDA:<hit count>,<mangled function name>
FNF:<functions found>   FNH:<functions hit>
DA:<line>,<hit count>                  # line execution count
LF:<lines found>        LH:<lines hit>
end_of_record
```

(No `BRDA`/branch data in Tilck's reports — branch coverage is absent
from the HTML too, so the viewer ignores branches.)

Everything the report shows is derived from these records plus the
source files named by `SF:`:

- **Line coverage** of a file: `LH/LF`; a line is *covered* if its `DA`
  count > 0, *uncovered* if `DA` count == 0, *non-instrumented* if it has
  no `DA` record.
- **Function coverage** of a file: `FNH/FNF`; a function is *hit* if its
  `FNDA` count > 0.
- **Directory** = the directory portion of the `SF:` path, made relative
  to the report root (the longest common prefix genhtml strips). The
  report lists directories as a **flat list of full relative paths**
  (e.g. `kernel/mm`, `common/3rd_party`), NOT a tree.

## 1. Coverage rate → color classification (used everywhere)

Three buckets, by percentage `p` (genhtml defaults):

| Bucket | Condition   | HTML color (bg) | TUI color (proposed) |
|--------|-------------|-----------------|----------------------|
| Hi     | `p >= 90`   | green `#A7FC9D` | green                |
| Med    | `75 <= p<90`| yellow `#FFEA20`| yellow               |
| Lo     | `p < 75`    | red `#FF0000`   | red                  |

A directory/file with **0 instrumented items** is shown as `0.0 %` (Lo).
Percentages display with one decimal and a non-breaking space before `%`
(`64.3 %`). The rate is `round(100 * hit / total, 1)`; `total == 0`
renders as `0.0 %` (genhtml prints `-` only with `--no-...`; this report
shows `0.0 %`).

## 2. Common page chrome (all four views)

Every page has the same header block:

1. **Title bar** — text `LCOV - code coverage report` on a blue
   (`#6688D4`) bar, full width.
2. **Ruler** — thin horizontal separator (a 3px image in HTML).
3. **Info grid** (2 columns of label/value on the left, a coverage
   sub-table on the right):
   - `Current view:` → breadcrumb (see §6).
   - `Test:` → `coverage.info` (the tracefile's `TN:`, empty here so the
     filename is used).
   - `Date:` → generation timestamp `YYYY-MM-DD HH:MM:SS`.
   - Right sub-table, header row `Hit | Total | Coverage`, then:
     - `Lines:` row → hit, total, `p %` (cell colored Lo/Med/Hi).
     - `Functions:` row → hit, total, `p %` (cell colored Lo/Med/Hi).
4. **Ruler**.
5. **Footer** (bottom of every page) — `Generated by: LCOV version 1.15`.

The header values are **per-view**: on the top page they are the grand
totals; on a directory page the directory's totals; on a file page the
file's totals.

## 3. View A — Directory overview (top-level `index.html`)

The landing view. A centered table:

- **Column headers**: `Directory` | `Line Coverage` (spans 3 sub-cols) |
  `Functions` (spans 2 sub-cols).
- Each header has a **sort control**:
  - `Directory` → sort by name (this is the default order; on the
    name-sorted page the control is inert/greyed).
  - `Line Coverage` → link to `index-sort-l.html` (sort by line %).
  - `Functions` → link to `index-sort-f.html` (sort by function %).
- **One row per directory** (flat full path), with:
  1. **Name** cell — the directory path, a link to that directory's
     file-list view.
  2. **Coverage bar** — a fixed-width bar: a colored segment
     (ruby/amber/emerald by the *line* rate bucket Lo/Med/Hi) of width
     `round(p)%` of the bar, the remainder in "snow" (empty). Width
     proportional to the line coverage %.
  3. **Line %** — `p %`, cell colored Lo/Med/Hi.
  4. **Line count** — `hit / total`, cell colored Lo/Med/Hi.
  5. **Function %** — `p %`, colored Lo/Med/Hi.
  6. **Function count** — `hit / total`, colored Lo/Med/Hi.

Sort orders (three, each a separate HTML page; in the TUI a toggle):
- **by name** (ascending, default),
- **by line coverage** (`index-sort-l.html`),
- **by function coverage** (`index-sort-f.html`).
genhtml's coverage sorts are ascending by rate (lowest first). The
currently-active sort's header shows a different icon; the others are
links.

## 4. View B — File list (per-directory `index.html`)

Identical layout to View A, except:
- Title: `LCOV - coverage.info - <dir>`.
- `Current view:` breadcrumb: `top level - <dir>` (`top level` links
  back to View A).
- First column header is **`Filename`** (not `Directory`).
- Rows are the **files** in that directory; the name links to the
  source view (`<file>.gcov.html`).
- Same three sort orders (`index-sort-l.html` / `index-sort-f.html`
  inside the directory).
- Same bar + Lo/Med/Hi coloring + line/function counts per file.

## 5. View C — Source view (`<file>.gcov.html`)

- Title: `LCOV - coverage.info - <dir>/<file>`.
- Breadcrumb: `top level - <dir> - <file> (source / functions)` where
  `top level` and `<dir>` are links and `functions` links to View D.
  In the source view the word `source` is plain text and `functions` is
  the link; in View D it is reversed.
- A monospaced listing with a heading line:
  `          Line data    Source code`
- **One row per source line**, three fields:
  1. **Line number** — right-aligned.
  2. **Hit count** — right-aligned; blank for non-instrumented lines,
     otherwise the `DA` execution count (e.g. `200133`). genhtml shows
     the raw count (it can also show `k`/`M` suffixes with `--...`, but
     this report shows the raw integer).
  3. **`:` separator then the source code** (tabs expanded; HTML entities
     like `&lt;`/`&amp;`/`&quot;` are the raw source `<`, `&`, `"`).
- **Per-line color**:
  - *covered* (`DA` > 0): light-blue bg `#CAD7FE` (`lineCov`).
  - *uncovered* (`DA` == 0): orange-red bg `#FF6230` (`lineNoCov`).
  - *non-instrumented* (no `DA`): no color.
- Each line is an anchor (`#<lineno>`) — targeted by function links and
  by the breadcrumb's deep links.

## 6. View D — Function list (`<file>.func.html`)

- Breadcrumb as in View C but with `source` linked and `functions`
  plain.
- Centered table, two columns:
  - **`Function Name`** (sort by name, default) — the (mangled) name,
    linking to the function's start line in the source view
    (`<file>.gcov.html#<line>`).
  - **`Hit count`** (sort by count → `<file>.func-sort-c.html`).
- **Per-row color** on the hit-count cell:
  - nonzero → light-blue `#DAE7FE` (`coverFnHi`),
  - zero → red `#FF0000` (`coverFnLo`).
- Two sort orders: by name (default), by hit count.

## 7. Navigation map (HTML links → TUI key bindings)

```
View A (dirs) --select dir-->            View B (files in dir)
View B (files) --select file-->          View C (source)
View C (source) --"functions"-->         View D (functions)
View D (functions) --"source"/select-->  View C (at function's line)
breadcrumb "top level" -->               View A
breadcrumb "<dir>" -->                   View B
each list view: cycle sort {name, line%, func%} / {name, count}
```

The HTML has no keyboard model (it is hyperlinks); the TUI must provide
keyboard navigation that reaches every destination above (see the plan
for the proposed key map) plus scrolling within long lists / long source
files, which the browser does natively.

- [ ] Directory overview: row per dir, name link

- [ ] Line % + line count cells, colored
- [ ] Function % + function count cells, colored

- [ ] Source line states: covered / uncovered / non-instrumented colors
- [ ] Function list: name + hit count, colored by zero/nonzero

- [ ] Threshold values Hi≥90, Med≥75 (configurable to match genhtml)

## 9. Out of scope (absent from Tilck's report, so not in the viewer)

- Branch coverage (`BRDA`) — not present.
- Multiple test cases / test descriptions (single `coverage.info`).
- "baseline"/differential coverage, ownership/date bins (genhtml 2.x
  features) — LCOV 1.15 report has none.
